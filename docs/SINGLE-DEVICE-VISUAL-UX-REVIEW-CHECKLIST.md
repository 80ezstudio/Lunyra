# Lunyra — Single Device Visual/UX Review Checklist

Use this in one uninterrupted device pass (10–15 min).

## 1) Splash feel (first launch)
- [ ] Loads quickly and feels premium (not generic)
- [ ] Starry/dark mood is calm, elegant, sleep-friendly
- [ ] Fade in/out feels smooth (no jank)
- [ ] Tap-to-skip works instantly

## 2) Setup clarity
- [ ] Screen feels clean and uncluttered
- [ ] Section order is intuitive: Resonance → Atmosphere → Color (contextual) → Session Length → More Options → Start
- [ ] Color options only show for Ambient Ring
- [ ] Start action is obvious and easy to reach

## 3) Premium visual quality (overall)
- [ ] Typography, spacing, and cards feel polished
- [ ] Glow is restrained (not tacky or flashy)
- [ ] Dark-room readability is good without being bright
- [ ] App feels cohesive, not pieced together

## 4) Starry Night behavior
- [ ] Star field is sparse and distant (not busy)
- [ ] Twinkle is subtle and non-distracting
- [ ] Shooting star appears around expected cadence (~45s, organic)
- [ ] Shooting star feels majestic/cinematic but still sleep-friendly
- [ ] No moment feels notification-bright

## 5) Ambient Ring quality
- [ ] Ring is visually dominant yet soft/luxurious
- [ ] Breathing pulse feels calm and intentional
- [ ] Color variants feel premium, not neon-gamer
- [ ] Motion remains restrained and soothing

## 6) Playback controls
- [ ] Single tap reliably reveals overlay
- [ ] Overlay auto-hides after ~3s
- [ ] Double tap pause/resume feels immediate
- [ ] End Session and Pause/Resume behave correctly

## 7) Bottom sheet smoothness
- [ ] Opens quickly without stutter
- [ ] Closing returns directly to live session
- [ ] Live setting changes apply predictably
- [ ] No disruptive transition or visual jump

## 8) Paid-app gut check (£1.69 target)
- [ ] Feels premium enough to pay for
- [ ] Experience feels intentional, calm, and high quality
- [ ] No obvious “cheap” visual/UX moments
- [ ] Would personally keep/use this at bedtime/focus sessions

## 9) Quick verdict
- [ ] PASS for next iteration
- [ ] PASS with minor fixes
- [ ] HOLD (list blockers before continuing)
